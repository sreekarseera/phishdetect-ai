## Step 1 — Stress-test the current model with real-sounding examples
Before touching the dataset, find out where it actually breaks. Write and run a quick script (or just use a Python shell) with **messages you make up yourself, not copied from `dataset.csv` or `generate_dataset.py`'s templates** — try real phishing email text you can find via search, and real-sounding personal/work messages. Example:
```python
import joblib
model = joblib.load("model/model.joblib")
for text in ["<your test message>", "<another one>"]:
    pred = model.predict([text])[0]
    prob = model.predict_proba([text])[0][pred]
    print(pred, round(prob, 2), text)
```
Note down anything it gets wrong or is only barely confident about (score close to 0.5).

## Step 2 — Add more/better examples
Using what you found in Step 1, add rows to `dataset.csv` (same `text,label` format) that specifically cover the gaps — e.g. if it missed a scam that didn't use urgent language, add more low-urgency scam examples. You can also extend `generate_dataset.py` with new template categories and re-run it (note: re-running it *appends* more rows — check `dataset.csv` doesn't end up wildly imbalanced between label 0 and 1 after).

## Step 3 — Retrain and re-verify
```bash
python3 train_model.py
```
Check the printed validation accuracy, then re-run your Step 1 stress test against the new `model/model.joblib` — confirm the cases it previously got wrong are now handled better, and it didn't lose accuracy on the original test cases.
